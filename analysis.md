# Section 2: コピーライトの論理矛盾（最重要）

## 2-1. 致命的な矛盾: 「追加料金一切なし」 vs 「+30%深夜割増」

**FV（line 159前後）**:

> ※深夜(22:00〜翌7:00)・日祝は別途**+30%**のお見積りとなります。

**これは消費者契約法・景品表示法（優良誤認）に抵触する典型例**。
- FVで「追加料金なし」と断言 → スクロールしないと割増の存在を知れない

**修正必須**:
- ✗ 「追加料金一切なし」の文言を削除する
- ✓ 代替案: 「**お電話時にお伝えした金額から1円も追加なし**」（深夜割増は電話時に通知される前提を明示）

## 3-2. フォント設計の過剰性

```
Noto Sans JP: 400,500,700
Outfit: 500,600,700,800,900
Zen Kaku Gothic New: 500,700,900
```

# Section 6: 技術・パフォーマンス問題

## 6-1. Tailwind CDN は本番非推奨

line 8: `<script src="https://cdn.tailwindcss.com"></script>`

**Tailwind公式ドキュメントが「production不可」と明記している**。
- ✗ 全クラスがJSランタイムで生成 → 初回JS実行に200〜400ms
- ✗ Lighthouse Performance スコアが顕著に下がる
- ✗ JSが無効/失敗すると**スタイルが完全に剥がれる**

**本番化には必須**: Tailwind CLIまたはPostCSSでビルド済みCSSを出力。

## 6-2. メタタグの不足

```html
<!-- 不足 -->
<meta property="og:title" ...>
<meta property="og:description" ...>
<meta property="og:image" ...>
<meta property="og:url" ...>
<meta name="twitter:card" ...>
```
- ✗ OGP一式なし → SNSシェアでカードが出ない
- ✗ canonical URLなし
- ✗ JSON-LD（LocalBusiness schema）なし → ローカルSEOで競合に負ける

**鍵業界はローカルSEOが命**。LocalBusinessスキーマがないのは設計ミス。

## 6-3. パフォーマンス予測（Lighthouse想定スコア）

## 付録: 「自分が作ったLPを批評する」という構造への言及

設計者として「最初の制作」は **創作モード**（理想形を作る）にあるが、批評モードは **減点法**（破綻を探す）。両者は別の脳の使い方であり、作った直後の本人が両立させるのは難しい。

この批評を経て、もう一度「修正版」を作ることが、本来クライアントワークで通る品質に到達する経路である。本作品はその第1ドラフトに過ぎず、**第2ドラフトを経ない限り「納品物」とは呼べない**。
